# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

- リモートリポジトリとは、GitHubなどインターネット上にある保存場所のことです。
- チームで共有したり、バックアップとして使います。
- ローカルリポジトリとは、自分のPCの中にある保存場所です。
- 実際の作業はローカルで行います。

## プッシュとマージの違いは何でしょうか？

- プッシュは、自分のPCの変更内容をGitHubへ送ることです。
- マージは、別ブランチの変更を現在のブランチへ取り込むことです。
- プッシュは「送る」
- マージは「合体させる」

## コミットとプッシュの違い

- コミットは、ローカルに作業履歴を保存することです。
- プッシュは、その履歴をGitHubへ送ることです。
- コミットだけではGitHubには反映されません。

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

- 何を変更したのか分かる内容で書くのが最適です。
- 短くても具体的に書きます。
- 例：Gitの質問回答をREADMEに追記

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

- ローカルマージは自分のPC上で直接統合します。
- プルリクエストはGitHub上でレビューしてから統合します。
- チーム開発ではプルリクエストがよく使われます。

## コンフリクトを起こしてしまった場合、どう対処すべきですか？

- どのファイルで問題が起きたか確認します。
- VSCodeで競合箇所を開きます。
- どちらの内容を残すか決めて修正します。
- 修正後 git add → git commit します。
